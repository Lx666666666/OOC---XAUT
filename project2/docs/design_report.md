# 任务2设计说明：网吧管理系统

## 一、系统目标

本项目是《OOP课程设计任务书》中任务2的样品实现。系统使用 C++17 和 EasyX4MinGW 开发图形界面，使用面向对象思想封装数据管理与界面交互。

## 二、类与数据结构

- `Record`：表示一条业务记录，包含编号、字段值和状态。
- `Manager`：负责记录的新增、修改、删除、读取、保存和排序。
- `GuiApp`：负责窗口绘制、按钮响应、输入对话框和状态提示。
- `std::vector<Record>`：保存所有业务记录。

## 三、文件流

数据保存在：

```text
data/records.csv
```

程序启动时自动读取，新增、修改、删除和特殊操作后自动保存。

## 四、任务要求对应

任务要求：输入顾客、输出顾客信息、修改顾客信息、按会员类型和上机时间结账。

样品中已实现基础增删改查，并加入 `顾客结账` 功能用于体现该题目的业务特点。

## 五、可扩展方向

- 增加查询框和按字段筛选。
- 增加更严格的数据校验。
- 增加打印报表或导出功能。
